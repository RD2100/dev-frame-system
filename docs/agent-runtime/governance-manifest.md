## Local Override Policy

```yaml
local_overrides:
  allowed:
    - project_name              # Target project name
    - project_description       # What this project does
    - local_paths               # Filesystem paths specific to this project
    - local_tool_versions       # Tool versions installed on this machine
    - local_capabilities        # Capabilities added locally (must register, not verified by default)
  forbidden:
    - weaken_p0_rules           # Changing "must" to "should", "required" to "recommended"
    - remove_gate_0             # Removing or disabling Gate 0 from workflow
    - disable_veto_contract     # Removing execute agent veto rights
    - widen_auto_trigger        # Increasing SADP trigger threshold to avoid governance
    - change_protected_files    # Removing files from protected list
    - modify_fallback_policy    # Weakening fallback constraints
```

## Drift Detection

On every session start, the agent must:

1. Compare protected section hashes against manifest
2. If any hash differs 闂?flag `governance_drift`
3. Check for unregistered local overrides 闂?flag `unauthorized_mutation`
4. Drift found in P0_RULES or GATE_0 闂?**pause execution, escalate to human**

```yaml
drift_action:
  protected_hash_changed: escalate
  unregistered_override: reject
  local_capability_added_without_registration: warn_and_register
  non_protected_file_changed: note_only
```
